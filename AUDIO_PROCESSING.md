# 音響信号処理および実験制御仕様（研究用）

本ドキュメントは、百人一首音声識別実験における音声刺激の調整手順および、提示タイミングの制御ロジックを記録したものである。

## 1. 音響提示の基本方針
本実験では、以下の2点を両立させるため、音声ファイルを物理的に加工しない「非破壊的オーバーラップ再生」を採用している。

1.  **時間的公平性**: 合図音（序歌）の読み終わりから、課題音声の実際の「発声開始（Onset）」までを正確に 1,000ms に固定する。
2.  **音響的連続性**: デジタル的な無音（無音区間）を作らず、録音環境由来の背景雑音（Ambient Noise）を維持することで、自然な空気感を保つ。

## 2. 音声解析および計測手法（Methods）
各音声刺激の「真の開始点（Onset）」を特定するため、以下の信号処理フィルタを適用した上で自動検知および手作業による精査を行った。

### 解析用プリプロセス
1.  **ハイパスフィルタ (100Hz)**: 低周波ノイズ（空調音・振動）の除去。
2.  **スペクトルノイズ抑制 (FFTベース)**: ホワイトノイズの低減。
3.  **ノイズゲート (-30dB閾値)**: 読み上げ直前の吸気音（ブレス）やリップノイズを無音化。

上記処理により、背景雑音に埋もれた微細な発声の立ち上がりをミリ秒単位で特定した。

## 3. シームレス再生アルゴリズム
プログラム側で以下の計算を行い、再生タイミングを動的に制御している。発声開始は常に **序歌読み終わりから 1,000ms 後（`voiceOnsetTime = now + 3.0s`）** に固定される。

### 3-1. Onset による2ケース分岐

各課題音声ファイルの先頭無音（`onset`）の長さに応じて、再生開始オフセットとタイミングを切り替える。

| 変数 | 意味 |
|:---|:---|
| `onset` | 課題音声ファイル先頭から発声が始まるまでの無音秒数（`kimariji.json` より取得） |
| `stimulusOffset` | 課題音声ファイルの再生開始位置（秒）。ファイル途中から再生する場合に使用 |
| `stimulusStartTime` | 課題音声の再生を予約する AudioContext 上の絶対時刻 |
| `voiceOnsetTime` | 発声が実際に始まる AudioContext 上の絶対時刻（`= stimulusStartTime + onset - stimulusOffset`、常に `now + 3.0s`） |
| `naturalCueEndTime` | ループなしで序歌ファイルが自然に終わる時刻（`= now + duration - cueOffset`） |

**ケースA（onset < 1.0s）**: 課題音声の先頭無音が足りないため、序歌末尾ノイズで補う

```
now       now+2.8s     now+2.8+(1-onset)s  now+3.8s
|---------|------------|-------------------|
 序歌(2.8s)  末尾ノイズ(ループ中)  課題音声再生開始      発声開始
stimulusOffset = 0（先頭から）
stimulusStartTime = now + 2.8 + (1 - onset)
```

**ケースB（onset ≥ 1.0s）**: 課題音声に十分な無音があるため、余分な無音をスキップして序歌直後に開始

```
now       now+2.8s                     now+3.8s
|---------|[onset-1s][       1s       ]|
 序歌(2.8s)  ←スキップ→  課題音声(途中から)   発声開始
stimulusOffset = onset - 1.0（余分な無音をスキップ）
stimulusStartTime = now + 2.8
```

### 3-2. 条件付きループ
- `cueSource.loop` は `stimulusStartTime > naturalCueEndTime` のときのみ `true` に設定する。
- ループ区間は `loopStart = cueSpeechEnd`、`loopEnd = duration`（読み終わり以降の末尾ノイズのみをループ）。
- 末尾ノイズが十分長い場合（例：木本 0.790s）は多くのケースでループ不要となり、末尾ノイズが自然に終わったあと課題音声の無音区間が継続する。

### 3-3. 合図音のフェードアウトと先読み
- **クロスフェード停止**: `stimulusStartTime` を起点に `crossfadeDuration = min(0.3s, onset)` かけて `cueGain` をゼロへ線形ランプし、その後 `activeCueSource.stop()` する。課題音声の無音区間への汚染を防ぎつつ、音響的な連続性を維持する。
- **高精度スケジューリング**: Web Audio API の `ctx.currentTime` を使用し、ネットワーク遅延やブラウザの処理待ちに影響されないミリ秒精度の予約再生（Sample-accurate timing）を実装している。
- **先読み (Preloading)**: 合図音が流れている間に次の課題音声データをメモリ上にデコード済みとして保持し、再生開始時の遅延をゼロにしている。

## 4. 論文用 Methods 記載例

> **音響提示のシームレス化と時間的整合性の確保**
> 実験中のデジタル的な無音状態による参加者の心理的影響を抑制し、かつ合図音終了から発声開始までの時間間隔を1,000msに厳密に固定するため、非破壊的なオーバーラップ再生手法を採用した。
> まず、全音声刺激について、発声の音響的な開始点（Onset）をミリ秒単位で事前に特定した。次に、各刺激のOnset時間に応じて2通りの方法でタイミングを算出した。Onset時間が1,000ms未満の刺激については、ファイル先頭から再生を開始し、合図音の末尾残響区間をループして不足分を補うことで発声開始を合図終了から1,000ms後に固定した。Onset時間が1,000ms以上の刺激については、ファイル先頭の余剰無音区間（Onset − 1,000ms分）をスキップして再生を開始することで、合図音終了直後から課題音声の再生を始め、同様に発声開始を合図終了から1,000ms後に揃えた。課題音声の発声開始と同時に合図音を停止することで、音響的な空気感（Atmosphere）を一定に維持しながら、全刺激において時間的な公平性と音響的なシームレス性を両立させた。
