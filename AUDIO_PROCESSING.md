# 音響信号処理および実験制御仕様（研究用）

本ドキュメントは、百人一首音声識別実験における音声刺激の調整手順および、提示タイミングの制御ロジックを記録したものである。

## 1. 音響提示の基本方針
本実験では、以下の2点を両立させるため、音声ファイルを物理的に加工しない「非破壊的オーバーラップ再生」を採用している。

1.  **時間的公平性**: 合図音（序歌）の読み終わりから、課題音声の実際の「発声開始（Onset）」までを正確に 1,000ms に固定する。
2.  **音響的連続性**: デジタル的な無音（無音区間）を作らず、録音環境由来の背景雑音（Ambient Noise）を維持することで、自然な空気感を保つ。

## 2. 音声解析および計測手法（Methods）
各音声刺激の「真の開始点（Onset）」を特定するため、以下の信号処理フィルタを適用した上で自動検知および手作業による精査を行った。

### 解析用プリプロセス
1.  **ハイパスフィルタ (100Hz)**: 低周波ノイズ（空調音・振動）の除去。
2.  **スペクトルノイズ抑制 (FFTベース)**: ホワイトノイズの低減。
3.  **ノイズゲート (-30dB閾値)**: 読み上げ直前の吸気音（ブレス）やリップノイズを無音化。

上記処理により、背景雑音に埋もれた微細な発声の立ち上がりをミリ秒単位で特定した。

## 3. シームレス再生アルゴリズム
プログラム側で以下の計算を行い、再生タイミングを動的に制御している。

- **オーバーラップ処理**:
  合図音（序歌）の再生を停止させず、課題音声を重ねて再生開始することで、背景雑音の連続性を確保する。
- **アンビエント・ループ (Ambient Loop)**:
  合図音の末尾（読み上げ終了後の残響部分）を、課題音声の**発声開始点**までループ再生し続けることで、デジタル的な無音状態を物理的に排除する。合図音が止まるタイミングは `activeCueSource.stop(stimulusStartTime + onset)` で制御し、課題音声の冒頭ノイズ区間も合図の残響でカバーする。
- **高精度スケジューリング (Scheduled Playback)**:
  Web Audio API の `ctx.currentTime` を使用し、ネットワーク遅延やブラウザの処理待ちに影響されないミリ秒精度の予約再生（Sample-accurate timing）を実装している。
- **先読み (Preloading)**:
  合図音が流れている間に次の課題音声データをメモリ上にデコード済みとして保持し、再生開始時の遅延をゼロにしている。

## 4. 論文用 Methods 記載例

> **音響提示のシームレス化と時間的整合性の確保**
> 実験中のデジタル的な無音状態による参加者の心理的影響を抑制し、かつ合図音終了から発声開始までの時間間隔を1,000msに厳密に固定するため、非破壊的なオーバーラップ再生手法を採用した。
> まず、全音声刺激について、発声の音響的な開始点（Onset）をミリ秒単位で事前に特定した。次に、制御プログラムにおいて、合図音の読み上げ終了時点を起点とし、1,000msから各課題音声固有のOnset時間を差し引いた値を待機時間として動的に算出した。課題音声の提示開始時には、合図音の背景雑音（Ambient Noise）を停止させずに継続再生させることで、音響的な空気感（Atmosphere）を一定に維持した。これにより、全ての刺激において時間的な公平性と音響的なシームレス性を両立させた。
