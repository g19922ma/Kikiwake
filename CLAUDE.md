# Kikiwake プロジェクト指示

## ファイルを読むタイミング
質問内容に関係するファイルだけを読むこと。全部読まない。

| 質問の内容 | 読むファイル |
|-----------|------------|
| データ保存・GAS・スプレッドシート関連 | `gas/コード.js` |
| 画面レイアウト・HTML構造関連 | `index.html` |
| ロジック・動作・バグ関連 | `script.js`（必要なら`index.html`も） |
| スタイル・見た目関連 | `style.css` |
| フロントとGASの連携関連 | `script.js` + `gas/コード.js` |

## コード変更時のprompt.md更新ルール
実験仕様に影響するコード変更（ボタン操作・タイミング・UI・データ保存など）を行ったときは、対応する内容を `prompt.md` にも反映すること。

## ユーザーが「しばらく操作できない」と言ったとき
確認・承認が必要な事項をリストに貯めておくこと。ユーザーが戻ったら一覧を提示し、一気に確認・実行する。
