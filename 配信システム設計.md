# 配信システム設計：Kikiwake × YouTube競技かるた配信

## 概要・目的

競技かるた選手6名のKikiwakeデータを活用し、YouTube配信の解説品質を向上させる。
試合中にリアルタイムで「この選手はこの歌が得意/苦手」「読まれた歌での有利不利」を可視化することを目的とする。

---

## 現在の実装（2026-03-12時点）

### broadcast_overlay.html（2選手デモ版）
- **ファイル**: `Kikiwake/broadcast_overlay.html`
- **対象選手**: maruyama1（青 #58a6ff）、kurihara-pilot1（緑 #3fb950）
- **データ**: 全100首のt_prime統計をインライン埋め込み（`songs_data.json` 由来）

**3タブ構成:**

| タブ | 内容 |
|---|---|
| 📺 配信オーバーレイ | 歌を選択 → 2選手のリアルタイム比較カード表示 |
| 📊 全100首グラフ | 2選手の全100首平均RTを横並び棒グラフで比較 |
| 👤 選手プロファイル | 得意/苦手ベスト5、全体統計 |

**配信オーバーレイの機能:**
- 左パネル: 全100首を反応速度順に一覧（速い歌が上）、検索フィルタ付き
- 中央: 選曲で選手比較カード表示（スピードバー、ベストタイム、SD、有利バッジ）
- 解説テキスト自動生成（歌の特性・選手の得意度に応じた文章）
- 難易度表示（⭐ 1〜5）

## データ構造

### songs_data.json
```json
{
  "17": {
    "kimariji": "ちは",
    "cv": 25.3,
    "mean_all": 320,
    "maruyama1": { "mean": 280, "sd": 45, "best": 210, "n": 6 },
    "kurihara":  { "mean": 350, "sd": 70, "best": 290, "n": 6 },
    "difficulty": 1
  }
}
```

**difficulty の計算方法:**
- `mean_all`（全参加者平均）の百分位ランクから算出
- 1 = 最速（よく知られた歌）〜 5 = 最遅（難しい歌）

## 6選手版への拡張

### ステップ1: データ収集
1. 各選手に Kikiwake を実施（rep2-4 推奨、51分）
2. `Kikiwake Data.xlsx` の `trials` シートに追記される
3. 参加者IDは選手名ベースで統一（例: `yoshida1`, `nishida1`）

### ステップ2: songs_data.json の再生成
既存の `analyze_rep_reduction.py` と同様に、t_prime中央値（または平均）を歌ごとに集計して JSON に書き出す。
現在の2選手構造に選手を追加するだけ:

```json
"17": {
  "kimariji": "ちは",
  "mean_all": 310,
  "maruyama1": { ... },
  "kurihara":  { ... },
  "yoshida1":  { "mean": 260, "sd": 38, "best": 205, "n": 3 },
  "nishida1":  { "mean": 295, "sd": 52, "best": 240, "n": 3 }
}
```

### ステップ3: broadcast_overlay.html の修正箇所

1. **PLAYERS 配列**に選手名・表示名・カラーコードを追加
2. **SONGS 定数**の各歌オブジェクトに選手データを追加
3. **試合選択UI**を追加（「対戦カード: 選手A vs 選手B」を選択すると比較表示が切り替わる）
4. 解説テキスト生成ロジックを6選手に対応させる

## OBS統合方法

### 基本セットアップ
1. OBS → ソースを追加 → **ブラウザ**
2. URL: `file:///Users/[ユーザー名]/Documents/Kikiwake/broadcast_overlay.html`（ローカルファイル）
3. 幅: 1920px、高さ: 1080px（配信解像度に合わせて調整）
4. カスタムCSS: 不要（スタイルはファイル内に全て含む）

### 配信フロー
1. 試合前に対戦カードを設定（選手A vs 選手B）
2. 解説者の手元PC でブラウザを開く（または OBS プレビューを参照）
3. 読手が歌を読む直前/直後に対応する歌をクリック
4. 解説者がオーバーレイを見ながら「この歌は〇〇選手が得意で...」とコメント

### 将来拡張: リアルタイム自動更新
- 現在は手動で歌を選択
- `selectSong(id)` がグローバルに公開されているため外部からも呼び出し可能
- 将来的には WebSocket または Broadcast Channel API で歌IDを外部送信して自動連動可能

## 注意事項

- **現データはパイロット参加者のもの**。実際の競技選手はt_primeが大幅に短い可能性が高い（特に1字決まり）
- **rep1は除外を推奨**。学習バイアスが大きく（kurihara: −38.8ms）、速度指標として不適切
- **セッション間学習に注意**。kurihara-pilot1では2セッション間でrep3が133ms改善。複数日収録の場合、後のセッションの値を採用するか平均化するか方針を決めること
- **収録推奨条件**: 1セッション完結（rep2-4, 51分）、PC＋イヤホン、静かな環境
